╔══════════════════════════════════════════════════════════════╗
║         GUIA DE CONEXIONES - DRON CASERO (DIY)              ║
║         Arduino UNO + ESP32-CAM + MPU6500 + 4 ESCs          ║
╚══════════════════════════════════════════════════════════════╝

Leete esto COMPLETO antes de conectar nada. Si conectas algo mal,
puedes quemar el Arduino, el ESP32 o los ESCs.




1.  QUE COMPONENTES TIENES
    ═══════════════════════

    [A] Arduino UNO          → el cerebro, controla el vuelo
    [B] ESP32-CAM            → camara + WiFi (recibe ordenes del movil)
    [C] MPU6500              → sensor de posicion (sabe si el dron esta inclinado)
    [D] 4x ESC + Motor       → cada ESC controla un motor (XW 30A segun dices)
    [E] Bateria 3S LiPo      → 11.1V (12.6V cuando esta llena)
    [F] BEC del ESC          → convierte 11.1V de la bateria a 5V para Arduino

    OPCIONAL:
    [G] 2 LEDs rojos + 2 resistencias 220ohm  → luces de status

3.  PASO A PASO - CONEXIONES
    ═════════════════════════════

    ANTES DE EMPEZAR:
    - NO conectes la bateria hasta que todo este conectado y revisado
    - Usa cables de colores distintos para no confundirte
    - Recomendado: rojo = positivo (+), negro = negativo (-/GND),
      blanco/azul = senal/datos


    3.1  ALIMENTACION (5V)
    ─────────────────────────

    Solo UNO de los 4 ESCs tiene el cable rojo del BEC conectado.
    Los otros 3 ESCs deben tener su cable rojo DESCONECTADO (suelto
    o aislado con cinta). Si conectas los 4 BEC rojos juntos, se
    queman.

        ESC1 (el que eliges como "master"):
           Cable rojo (BEC 5V) ─── 5V del Arduino
           Cable negro ─────────── GND del Arduino
           Cable blanco (senal) ── D3 del Arduino

        ESC2, ESC3, ESC4:
           Cable rojo ──────────── NADA (aislado con cinta)
           Cable negro ─────────── GND del Arduino
           Cable blanco (senal) ── D5, D6, D9 respectivamente

        ⚠ IMPORTANTE: Todos los cables negros (GND) de los 4 ESCs
          DEBEN ir al GND del Arduino. El GND debe ser compartido
          entre todo el sistema: Arduino, ESCs, ESP32, MPU6500.


    3.2  MPU6500 (sensor de posicion)
    ───────────────────────────────────

        MPU6500 VCC ─── 5V del Arduino  (o 3.3V segun tu modulo)
        MPU6500 GND ─── GND del Arduino
        MPU6500 SDA ─── A4 del Arduino
        MPU6500 SCL ─── A5 del Arduino

        ⚠ Si tu modulo MPU6500 funciona a 3.3V, NO lo conectes a 5V
          o lo quemas. Revisa la etiqueta del modulo.


    3.3  ESP32-CAM (WiFi + camara)
    ─────────────────────────────────

        ATENCION: El Arduino UNO usa 5V en sus pines. El ESP32-CAM
        usa 3.3V. Si conectas D1(TX) del Arduino directo al RX del
        ESP32, quemas el pin del ESP32. Por eso usamos un DIVISOR
        DE VOLTAJE.

        ┌─── DIVISOR DE VOLTAJE (importante, no te lo saltes) ──────┐
        │                                                           │
        │  Arduino D1(TX) 5V ─── R1=1k ohm ──┬── R2=2.2k ohm ── GND │
        │                                     │                     │
        │                                    ┌┴┐                    │
        │                                   ESP32-CAM RX (GPIO3)    │
        │                                   (3.3V)                  │
        └───────────────────────────────────────────────────────────┘

        ESP32-CAM GPIO1 (TX) ─── directo a Arduino D0(RX)
        ESP32-CAM GPIO3 (RX) ─── a traves del divisor (ver arriba)
        ESP32-CAM 5V ─────────── 5V del BEC (puede ir al mismo 5V
                                 que el Arduino)
        ESP32-CAM GND ────────── GND comun

        Ademas, pon un capacitor de 100uF + uno de 0.1uF en
        paralelo entre 5V y GND del ESP32 (lo mas cerca posible
        del ESP32). Esto evita que la camara cause apagones.

        ⚠ No conectes el ESP32-CAM sin estos capacitores. La camara
          consume picos de corriente y puede reiniciar el Arduino.


    3.4  LEDS de status (opcional)
    ────────────────────────────────

        LED1 (rojo, armado):  D10 ─── resistencia 220ohm ─── (+)LED(-) ─── GND
        LED2 (rojo, latido):  D13 ─── resistencia 220ohm ─── (+)LED(-) ─── GND

        ⚠ La resistencia es OBLIGATORIA. Sin ella quemas el LED
          o el pin del Arduino.


    3.5  DIVISOR para medir bateria
    ─────────────────────────────────

        Para que el Arduino sepa cuanta bateria queda, usas un
        divisor de voltaje en A0:

        Bateria (+) ─── R1=100k ohm ──┬── R3=33k ohm ── GND
                                       │
                                      A0 (Arduino)

        La formula: Vmedido = Vbateria * 33k / (100k + 33k)
        Esto convierte 12.6V (bateria llena) a unos 3.1V, seguro
        para el Arduino.





4.  TABLA RAPIDA DE PINES
    ════════════════════════

    ARDUINO UNO         CONECTA A                       QUE HACE
    ───────────         ─────────                       ─────────
    D0 (RX)             ESP32-CAM GPIO1 (TX)            Recibe datos del ESP32
    D1 (TX)             ESP32-CAM GPIO3 (RX)            Envia datos al ESP32
                        (via divisor 1k+2.2k)
    D3                  ESC1 (senal blanca)              Motor 1 (FR, gira CW)
    D5                  ESC2 (senal blanca)              Motor 2 (RL, gira CW)
    D6                  ESC3 (senal blanca)              Motor 3 (FL, gira CCW)
    D9                  ESC4 (senal blanca)              Motor 4 (RR, gira CCW)
    D10                 LED1 (via 220ohm)                Luz roja = armado
    D13                 LED2 (via 220ohm)                Luz roja = heartbeat
    A0                  Divisor bateria (100k+33k)       Mide voltaje bateria
    A4 (SDA)            MPU6500 SDA                      I2C datos
    A5 (SCL)            MPU6500 SCL                      I2C reloj
    5V                  BEC 5V (solo 1 ESC) +            Alimentacion 5V
                        ESP32-CAM 5V + MPU6500 VCC
    GND                 GND de TODO                      Tierra comun

    ESP32-CAM           CONECTA A
    ──────────           ─────────
    GPIO1 (TX)          Arduino D0 (RX)
    GPIO3 (RX)          Divisor 1k+2.2k → Arduino D1 (TX)
    5V                  5V general (desde BEC)
    GND                 GND comun

    MPU6500             CONECTA A
    ───────             ─────────
    VCC                 5V o 3.3V (segun modulo)
    GND                 GND
    SDA                 Arduino A4
    SCL                 Arduino A5




5.  ERRORES COMUNES
    ═══════════════════

    ❌ Conectar los 4 cables rojos de los ESCs al Arduino
       → Los BEC pelearan entre si y se queman.
       → Solucion: solo 1 cable rojo conectado, los otros 3
         aislados con cinta.

    ❌ Conectar D1(TX) del Arduino directo al ESP32
       → El Arduino manda 5V al ESP32 que solo aguanta 3.3V.
       → Quemas el GPIO3 del ESP32.
       → Solucion: divisor 1k+2.2k entre D1 y RX del ESP32.

    ❌ No poner capacitores en el ESP32-CAM
       → La camara consume picos de corriente y el ESP32 se
         reinicia constantemente.
       → Solucion: 100uF + 0.1uF entre 5V y GND del ESP32.

    ❌ Conectar la bateria al reves
       → Los ESCs tienen proteccion, pero mejor revisa.
       → Rojo = positivo (+), Negro = negativo (-).

    ❌ GND no compartido entre todos los componentes
       → Las senales no llegan bien, el MPU6500 da lecturas
         raras, el ESP32 no se comunica.
       → Solucion: TODOS los GND conectados entre si.

    ❌ No calibrar los ESCs antes del primer vuelo
       → Los motores no arrancan todos al mismo tiempo.
       → Solucion: sube el throttle a 2000 cuando conectas la
         bateria (los ESCs emiten unos tonos).




6.  PUERTO SERIAL (para programar y probar)
    ═══════════════════════════════════════════

    Cuando conectas el Arduino UNO al PC via USB:
    - Arduino IDE: Tools → Port → COMx (Arduino Uno)
    - Serial Monitor: 57600 baud (NO 9600)
    - El Arduino usa 57600 para hablar con el ESP32 y contigo

    El ESP32-CAM NO se programa por USB directo (no tiene puerto
    USB). Necesitas un convertidor FTDI o CP2102:
    - FTDI TX → ESP32 RX (GPIO3)
    - FTDI RX → ESP32 TX (GPIO1)
    - FTDI GND → ESP32 GND
    - FTDI 5V → ESP32 5V
    - Poner GPIO0 a GND durante la conexion (modo descarga)
    - Quitar GPIO0 de GND y resetear para ejecutar




7.  ORDEN DE ENCENDIDO
    ═════════════════════

    1. Conecta el Arduino al PC (USB) - se programa solo
    2. Desconecta el USB del Arduino
    3. Conecta la bateria 3S LiPo a los ESCs (todos juntos)
       → Los ESCs emiten unos tonos (calibracion)
       → El BEC del ESC alimenta el Arduino y el ESP32
    4. Espera a que el ESP32 arranque su WiFi (unos segundos)
    5. Conectate desde el movil a la WiFi "Drone_ESP32"
    6. Abre navegador en 192.168.4.1
    7. Mueves los sliders para controlar el dron

    ⚠ NUNCA conectes el USB y la bateria al mismo tiempo si el
      PC no esta aislado. Puede quemar el puerto USB.
